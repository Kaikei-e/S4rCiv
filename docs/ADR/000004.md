# ADR-000004: kokkai snapshot の内部ミラー保持と UI 公開範囲を著作権法上の根拠で確定する

## Status

Accepted

## Date

2026-06-03

## Affected Services

- collector (Go) — snapshot を圧縮 bytes でミラー保持（情報解析目的）
- api (Go) — query response に出典 + 参照用 permalink を常時併記
- web (SvelteKit) — 議員発言の全文表示は可、個人別「発言集成」UI を作らない
- db (Postgres) — `observation.snapshot.bytes` にミラーを保持

## Context

[[000001]] は snapshot を「既定はミラー（圧縮 bytes）+ 例外的に external_ref」と決めつつ、Cons で
「全文の自前ミラーは NDL の DB 著作権（CORE_CONCEPT §7）と緊張し得るため、external_ref で回避する
運用判断が個別に要る」と保留した。国会会議録（kokkai）はまさにこの当該ケースである。あわせて
概念 §14.3（一次文書を自前ミラーするか Internet Archive リンク中心にするか）も未決だった。

加えて、UI でどこまで原文を出せるかが M1 の UX を直接左右する。snapshot は content-addressed な
圧縮 bytes（観測面の ground truth）であり、「内部に持つこと」と「UI で再配信すること」は別問題で
ある。これを混同せず、保持と公開それぞれの法的足場を確定する必要があった。

2026-06-03 に一次ソース（国会会議録検索 API / システムヘルプ、国立国会図書館ウェブサイトの
コンテンツ利用規約、著作権法、デジタル庁 公共データ利用規約 PDL1.0）を調査し、根拠を固めた。

## Decision

**内部ミラー保持を継続し、UI では議員発言の全文表示を許容する。両者を著作権法上の別根拠で
正当化し、「個人別の発言集成 UI を作らない」を設計不変条件として固定する。**

- **保持（内部ミラー）**: snapshot を圧縮 bytes で保持する（連鎖検証用に ground truth を自己完結
  させる）。法的根拠は**著作権法30条の4（電子計算機による情報解析）**。NDL も API ヘルプで情報
  解析を許諾不要の権利制限として明示している。
- **公開（UI 全文表示）**: 議員（= NDL 職員以外）の発言は**著作権法40条1項「公開して行われた
  政治上の演説又は陳述」**により方法を問わず利用できる。NDL 職員の発言および DB 編集物は
  **PDL1.0**（NDL ウェブサイトのコンテンツ利用規約が準拠）に従い、出典明記のうえ利用できる。
- **出典 + NDL 参照用 permalink を全表示・差分に常時併記する**（著作権法48条の出所明示、PDL1.0 の
  出典記載、DISCIPLINE §9）。改変・加工して表示する場合は編集主体を明示する（PDL1.0）。
- **設計不変条件: 個人別「発言集成」UI を作らない。** 40条1項但書「同一の著作者のものを編集して
  利用する場合」はこの権利制限の適用外であり、特定議員の発言だけを集成するアンソロジー／
  プロファイルは40条の保護を外れる。これは DISCIPLINE §4（反 doxxing・横断名寄せ禁止）と同一の
  線であり、**法と規律を同時に満たす単一ルール**として固定する。提示軸は会議 / 時系列 / 差分のみ。
- 検討した代替案と却下理由:
  - **external_ref のみ（ミラーしない）**: DB 著作権の緊張は避けられるが、ground truth が第三者
    依存になり（[[000001]] Cons）、原文が消えた後に連鎖を再検証できない。→ 却下。
  - **ミラーをそのまま全件バルク再配布**: DB 編集著作権と正面衝突する。→ 却下。API は個別レコード
    単位＋出典で出す。
  - **全文を一切 UI に出さず差分・スニペットのみ**: 40条1項が許す範囲を自ら狭め UX を損なう。→
    却下（差分は当然コンテキスト付きで出すが、全文表示も40条で正当化される）。

本決定は公的一次データのみ（§4-2）、出典明記の技術内蔵（§4-7）、コンテキスト保全（差分は周辺
コンテキスト＋全文リンク、§5-B / DISCIPLINE §7）、反 doxxing（§5-B / DISCIPLINE §4）の設計原則と
整合する。

## Consequences

### Pros

- ミラー保持により連鎖検証が自己完結し、第三者依存を排除できる（§13 再現性）。
- 議員発言の全文を法的にクリーンに UI 提示でき、UX を確保できる。
- 「個人別集成をしない」が40条但書（法）と反 doxxing（規律）の双方を同時に満たす単一の設計
  ルールになり、判断が一本化される。

### Cons / Tradeoffs

- 「個人別集成をしない」を UI / API 設計で恒常的に守る規律コストが生じる（speaker を属性として
  クエリできても、特定個人の発言を集成するビューは出さない、という線引きの徹底が要る）。
- DB 著作物の全件バルク再配布はしない、という運用上の自己制約が残る（API は個別レコード単位＋
  出典）。
- 法解釈は現行法・現行規約（2026-06 時点）に依存する。著作権法・NDL 規約・PDL の改定時は再評価が
  要る。
- `was_ocr` 由来の低信頼テキストを公開時にどう扱うか（概念 §14.5）は本 ADR の対象外で、別途未決の
  まま残る。
