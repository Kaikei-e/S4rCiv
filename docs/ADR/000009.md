# ADR-000009: 記名投票者名の抽出に Kagome（MeCab 相当）を採用し、適用を自由文に限定して IPADIC を AGPL 整合で同梱する

## Status

Accepted

## Date

2026-06-03

## Affected Services

- api (Go) — `internal/domain/legislative` の `vote_parser` が Kagome で投票者名を分割（`vote_names.go`）。`ExtractorVersion` を `0.2.0` に更新
- collector (Go) — 同じ `legislative` パッケージ経由で IPA 辞書をバイナリに埋め込む（projection が `ParseVotes` を駆動）
- 依存追加 — `github.com/ikawaha/kagome/v2`（MIT）＋ `github.com/ikawaha/kagome-dict/ipa`（MIT パッケージング／MeCab-IPADIC BSD系）。帰属は `services/api/THIRD_PARTY_LICENSES.md`

## Context

M4（[[000008]]）で giin-roster gateway と `vote_parser` の和文解析を組む際、**固定書式でない自由文**＝会議録の記名投票者名簿を、素朴な正規表現＋空白/読点分割で切り出していた。会議録末尾は OCR ノイズを含み、氏名の区切りが落ちて連結する（"逢沢一郎君青木ひとみ君…"）と、名簿が1名に潰れて announced count と一致せず、confidence が `high` から落ちて `needs_review` が増え、人手レビュー負荷を不必要に押し上げる。

レビューで「**ナイーブな字句解析をするなら、信頼できる MeCab 相当の Go ライブラリを入れる。ライセンスに気を付ける**」と要請された。制約は以下:

- サーバ本体 **AGPL-3.0**、**単一自己ホストバイナリ志向**（概念 §8）、pure Go・cgo 回避が望ましい。
- DISCIPLINE §6 — AI に**判定・評価・スコアリングをさせない**。形態素解析は判断ではなく**機械的・決定的な前処理**である必要がある。
- reproject 安全（§4-3）。再抽出は `ExtractorVersion` の bump による reproject で反映する（[[000004]] / immutable-design-guard #6）。

採用前に公式一次ソース（pkg.go.dev、各 GitHub README/LICENSE、Debian IpadicLicense、FSF Directory）でライブラリと**同梱辞書**のライセンスを確認した。

## Decision

**記名投票者名の抽出に Kagome v2（`github.com/ikawaha/kagome/v2`、pure Go・cgo なし）＋ `kagome-dict/ipa`（MeCab-IPADIC）を採用する。形態素解析の適用は自由文の人名抽出に限定し、固定書式の構造化フィールド（選挙区 "岡山1" → kucode 等）は決定的パースのまま据え置く。辞書は IPA のみとし UniDic は採らない。**

- **実装**: `vote_names.go` の `splitVoterNames` が、区切り（空白/読点）で粗分割した各 run を Kagome でトークナイズし、**君/さん が 接尾 POS のトークンの位置で分割**する。名前の一部としての「君」（接尾でない）では分割しないため、素朴な `strings.Split(s, "君")` が誤る連結ケースも正しく扱える。`vote_parser` の `extractNameBlock` がこれを呼ぶ。
- **決定性 / reproject 安全**: 固定辞書バージョンに対しトークナイズは決定的なので projection は reproject 安全。辞書・抽出器の更新は `ExtractorVersion`（`0.2.0`）の bump → reproject で反映する。
- **適用範囲の限定**: 官製の固定書式（"岡山1" → 県名＋番号 → `kucode`）に形態素解析を当てると誤分割し得るため、そこは決定的テーブル＋数字分割のまま。これは GeoJSON `kucode` との「同一導出」（[[000008]]）も保つ。
- **ライセンス（最重要）**: Kagome＝**MIT**、kagome-dict＝**MIT**、MeCab-IPADIC（`ipa` が埋め込むデータ）＝**ICOT/NAIST の BSD 系**（Debian *main*＝DFSG-free、著作権表示＋NO WARRANTY 保持が条件）。いずれも permissive で **AGPL-3.0 整合**。`THIRD_PARTY_LICENSES.md` に集約。
- **辞書は IPA のみ**: `uni` は未 import でバイナリに含めない。

検討した代替案と却下理由:

- **cgo の MeCab バインディング**（bluele/mecab-golang・shogo82148/go-mecab 等）: システムに MeCab＋辞書の別途インストールが要り**自己完結性を壊す**＋cgo 混入。→ 却下（単一バイナリ §8 と反する）。
- **形態素解析を入れず現状維持**: 最小だが OCR 連結に脆く、件数照合失敗で `needs_review` を増やす。→ 却下。
- **UniDic 辞書**: 精度は高いが **GPL-2.0-only OR LGPL-2.1-only OR BSD-3-Clause のトリプル**で GPL-2.0-only arm が AGPL-3.0 非互換、かつ数百MB級でバイナリ肥大。→ 却下（IPA 採用。将来採るなら **BSD-3-Clause を明示選択**する前提）。
- **形態素解析を名簿パース全体（選挙区含む）に適用**: 固定書式の誤分割リスク。→ 却下（自由文に限定）。

本決定は passive / read-only（解析は取得済みテキストの後処理）、**AI は判定しない**（形態素解析は機械的・決定的な前処理であって評価・スコアでない、§4-6 / DISCIPLINE §6）、reproject 安全（`ExtractorVersion`、§4-3）、単一バイナリ志向（pure Go・辞書埋め込み、§8。ただしバイナリ増分のコストを受容）、標準準拠でサイロを作らない（広く使われる OSS）の設計原則と整合する。

## Consequences

### Pros

- OCR で連結した投票者名簿を 接尾（君/さん）境界で正しく分割でき、announced count との一致率が上がって `needs_review` の過多/取りこぼしを減らせる。
- pure Go・cgo なし・辞書埋め込みで単一バイナリ志向を保つ。
- ライセンスが MIT＋BSD 系で AGPL 整合、`THIRD_PARTY_LICENSES.md` に明文化。
- 決定的＝reproject 安全。固定書式は決定的パースのままで誤分割を避ける。

### Cons / Tradeoffs

- **バイナリが +約15–20MB**（IPA 辞書埋め込み。実測 collector 32M / api 22M）。UniDic ならさらに桁違いに大きく、その回避が本決定の根拠の一つ。
- `domain/legislative` が形態素解析ライブラリに依存する（純粋・決定的の性格は保つが、初回辞書ロードに ~0.4s）。
- 政治家氏名の多くは IPADIC 未登録（未知語）で、分割境界は 接尾（君/さん）に依存する。君 を欠く名簿では従来の区切り分割にフォールバックする（ハイブリッド設計）。
- Kagome・辞書の更新時は `ExtractorVersion` の bump ＋ reproject が要る。
- IPADIC の NO WARRANTY/著作権表示の保持義務を、配布物で維持する運用が要る。
