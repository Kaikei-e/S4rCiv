# S4RCIV — Design Language

> ビジュアルデザインの規定。`CORE_CONCEPT`（プロダクト設計）・`CONTEXT.md`（用語集）と対。
> 既定はダークテーマ。目標は WCAG 2.2 AA。
> **v1（刷新）** — 中核は不変のまま、方向を `記録簿 (ledger)` に確定し、表現を磨いた。確定の根拠は [ADR-000020](../ADR/000020.md)。

---

## 0. v1 で確定したこと（要約）

v0 からの**中核は変えない**。確定したのは表現と供給方式。

- **方向は `記録簿 (ledger)` 単一。** エディトリアル（公報・新聞・記録簿の品位）。鋭い角・ヘアライン罫・呼吸する余白。受動・非判断の「淡々と記録する見張り」の態度を、運用画面の即応性ではなく記録簿の静けさで体現する。
- **`司令室 (room)` 方向は採らない。** 丸い角・サンセリフ見出し・浮くカードの代替案は本書に記述のみ残し、出荷しない（§1 末）。
- **状態色トーンは `calm`（彩度を抑制）単一。** v0 の彩度（`vivid`）は採らない（§3.3 末）。
- **セリフ（Zen Old Mincho）は Display／ページ H1／レコードタイトルに限定**（§4）。
- **フォントはセルフホスト（weight-controlled）。** 第三者へリクエストしない（§11）。
- **ランタイム軸は `[data-theme]`（dark/light）のみ。** `dir`/`tone`/`density` は焼き込み、切替は出荷しない。

---

## 1. 原則

これは願望ではなく制約。いずれかに反する表現は定義上まちがいで、作る前に矛盾を申告する。

1. **無彩色を基調にし、色は状態のためだけに使う。** UI の大半はニュートラルなグレースケール。色は状態（観測・変化・注意・消失）の伝達にのみ用い、平常時は彩度を持たせない。情報密度が高くても疲れにくくするための方針（High-Performance HMI の考え方）。
2. **純黒・純白を使わない。** 背景は暗いグレー（`#0A0D12` 系）、本文はオフホワイト（`#E9EDF4` 系）。コントラストが極端だと長時間の可読性が落ちるため。
3. **階層は影でなく明度で表す。** 暗い背景では影が効かないので、`surface-1`→`surface-3` の明度差と三段のヘアラインで層を作る。
4. **色だけに依存しない。** 状態は「色＋グリフ＋ラベル（必要に応じて形）」で多重に符号化する。差分は `+ / − / ~` の記号も併用（WCAG 1.4.1）。
5. **警告色は彩度を抑える。** 注意・消失などの色は、目立つが警報的になりすぎないトーン（`calm`）にする。実質的な変化だけを前面に出し、不要な通知で埋めない。
6. **出所と鮮度を常に表示する。** 各データに「最終取得時刻・出典・改ざん検証（ハッシュ連鎖）」を併記し、信頼性を可視化する。信頼を自称せず**検証可能性**を提示する。

### v1 で磨いたこと

- **エディトリアルな静けさ。** 公報・記録簿の品位。明朝の見出し（限定）、ヘアライン罫、呼吸する余白。
- **ヘアラインとサーフェスの質感。** 三段階のヘアラインと `surface-inset`/`surface-raise` で、影に頼らず明度とごく薄い面で層を作る。
- **タイポグラフィの階層。** 明確なスケール（`--fs-*`）、mono ラベル、tabular な数値。
- **状態色トーンの再調整。** 既定を `calm`（くすみ）に。警報的にならない落ち着いた彩度。

### 採らなかった代替案（出荷しない）

> 凍結は意味論（色＝状態／多重符号／純黒純白なし／provenance 常時／パネルが最小単位／数値は mono tabular）であって佇まいではない。佇まいは下記の通り**一つに確定**した。再導入には相応の実装と WCAG 再検証を要する（[ADR-000020](../ADR/000020.md)）。

- **`司令室 (room)` 方向** — 角丸 `6 / 10 / 14px`、見出しサンセリフ、明度で浮くカード。運用画面の一望性。**採らない**：即応的トーンが受動原則①と相反する。
- **`vivid` トーン** — v0 の彩度（changed `#E0A845` 等）。**採らない**：principle #5（彩度抑制）に対し calm を採る。

## 2. デザイントークン

役割で命名し、テーマ切替を一括で行う。CSS カスタムプロパティとして実装する。値は **dark 既定＋ `[data-theme="light"]`** の 2 系統のみ（`dir`/`tone`/`density` の軸は持たない）。

### Dark（既定・`calm`・`ledger`）

```css
:root,
[data-theme='dark'] {
  /* surfaces — 明度で階層、影は使わない */
  --canvas:        #0a0d12;
  --surface-1:     #0f141b;  /* パネル */
  --surface-2:     #161d26;  /* パネルヘッダ・一段上 */
  --surface-3:     #1e2732;  /* オーバーレイ・ポップオーバー */
  --surface-inset: #0b0f15;  /* 差分 well・コード */
  --surface-raise: #232e3a;  /* ホバーのごく薄い浮き */

  /* hairlines — 三段 */
  --hairline:   rgba(182, 202, 228, 0.07);  /* 最も繊細・行間 */
  --hairline-2: rgba(182, 202, 228, 0.14);  /* 標準・パネル輪郭 */
  --hairline-3: rgba(182, 202, 228, 0.24);  /* 強・編集罫 */

  /* text */
  --text-1:     #e9edf4;  /* 主要 */
  --text-2:     #abb6c7;  /* 副次 */
  --text-3:     #7d8799;  /* 補助・ラベル */
  --text-faint: #5a6473;  /* 最弱・装飾的メタ */

  /* interactive */
  --accent:      #5fb6ce;
  --accent-2:    #8fcfe2;
  --accent-weak: rgba(95, 182, 206, 0.12);
  --accent-line: rgba(95, 182, 206, 0.34);
  --focus:       #8fd6ea;
  --on-accent:   #04222c;  /* accent 背景上の文字 */

  /* status（calm）: graphic(塗り/枠) / text(濃い背景上の文字) */
  --st-nominal:  #4fa98a; --st-nominal-t:  #77c8a9;
  --st-info:     #5e97c9; --st-info-t:     #92bbdf;
  --st-changed:  #d2a24f; --st-changed-t:  #e7c483;
  --st-caution:  #d0814a; --st-caution-t:  #e7a877;
  --st-critical: #cc6b66; --st-critical-t: #e69089;

  /* data-viz（Okabe-Ito をダーク・calm 調整。形状と併用） */
  --dv-1:#5e97c9; --dv-2:#d2a24f; --dv-3:#4fb295;
  --dv-4:#b487c9; --dv-5:#d4796d; --dv-6:#93c061;

  /* shape（ledger） */
  --r-sm:2px; --r:3px; --r-lg:4px; --r-pill:999px; --rule:1px;

  /* spacing（4px グリッド） */
  --s1:4px; --s2:8px; --s3:12px; --s4:16px;
  --s5:24px; --s6:32px; --s7:48px; --s8:64px;

  /* type families */
  --font-sans:    'IBM Plex Sans JP','IBM Plex Sans',system-ui,sans-serif;
  --font-serif:   'Zen Old Mincho','Hiragino Mincho ProN','Yu Mincho','Noto Serif JP',serif;
  --font-mono:    'IBM Plex Mono',ui-monospace,SFMono-Regular,Menlo,monospace;
  --font-display: var(--font-serif);

  /* type scale */
  --fs-display:30px; --fs-h1:22px; --fs-h2:17px; --fs-h3:15px;
  --fs-body:15px; --fs-sm:13px; --fs-xs:12px; --fs-label:11px;
  --lh-body:1.65;

  /* breakpoints（§9.1） */
  --bp-sm:30rem; --bp-md:48rem; --bp-lg:55rem; --bp-xl:75rem;

  /* density（regular 既定。切替は出荷しない） */
  --row-pad:15px; --panel-pad:20px;
}
```

## 3. カラー

### 3.1 サーフェス（明度の段階）

| トークン | Hex (dark) | 用途 |
|---|---|---|
| `--canvas` | `#0A0D12` | ページ背景 |
| `--surface-1` | `#0F141B` | パネル |
| `--surface-2` | `#161D26` | パネルヘッダ・一段上の面 |
| `--surface-3` | `#1E2732` | オーバーレイ・ポップオーバー |
| `--surface-inset` | `#0B0F15` | 差分 well・コード（くぼみ） |
| `--surface-raise` | `#232E3A` | ホバーのごく薄い浮き |

### 3.2 テキスト・アクセント（コントラスト）

コントラスト目標: 本文 ≥ 4.5:1、大きい文字/非テキスト ≥ 3:1（WCAG 2.2 / 1.4.3・1.4.11）。**値は概算。確定時に実測ツールで再検証する。**

| トークン | Hex (dark) | on `--canvas` | 判定 |
|---|---|---|---|
| `--text-1` | `#E9EDF4` | ≈ 15:1 | AAA |
| `--text-2` | `#ABB6C7` | ≈ 8.5:1 | AAA |
| `--text-3` | `#7D8799` | ≈ 5:1 | AA（大はAAA） |
| `--text-faint` | `#5A6473` | ≈ 3:1 | 装飾的メタのみ（本文に使わない） |
| `--accent` | `#5FB6CE` | ≈ 8:1 | リンク/選択/フォーカス |

### 3.3 状態色（色＝意味・`calm`）

状態は**色＋グリフ＋ラベル**で必ず多重符号化する。

| 状態 | role | graphic | text | グリフ | 用途 |
|---|---|---|---|---|---|
| 安定 | nominal | `#4FA98A` | `#77C8A9` | `●` | 変化なし・正常・復活 |
| 観測 | info | `#5E97C9` | `#92BBDF` | `◉` | 新規取得・記録 |
| 変化 | changed | `#D2A24F` | `#E7C483` | `Δ` | 実質的な変更（中心となる状態色） |
| 注意 | caution | `#D0814A` | `#E7A877` | `▲` | 低信頼・要確認（OCR 由来等） |
| 消失 | critical | `#CC6B66` | `#E69089` | `⊘` | 公開記録の削除 |

- 変更（changed）= 琥珀＋`Δ` を基準色とする。
- 消失（critical）はくすんだ赤。目立つが警報的にしない。
- pill 実装: 背景 = graphic を低 alpha（≈13%）、枠 = graphic ≈0.42、文字 = text 変種、＋グリフ＋ラベル。
- **`vivid`（v0 彩度）は採らない**（§1 末）。calm を唯一の状態パレットとする。

### 3.4 データ可視化パレット

`--dv-1`〜`--dv-6`。Okabe-Ito を暗背景・calm 用に調整した色覚多様性配慮パレット。**グラフではマーカー形状・パターンを必ず併用**し、色だけに依存しない。暗背景は色の弁別にむしろ有利（IBM Carbon の知見）。色数は必要最小限に絞る。

| トークン | 形状（併用例） |
|---|---|
| `--dv-1` `#5E97C9` | ● 円 |
| `--dv-2` `#D2A24F` | ■ 四角 |
| `--dv-3` `#4FB295` | ▲ 三角 |
| `--dv-4` `#B487C9` | ◆ 菱形 |
| `--dv-5` `#D4796D` | ✚ 十字 |
| `--dv-6` `#93C061` | ★ 星 |

## 4. タイポグラフィ

3 つの書体を役割で使い分ける。供給はセルフホスト（§11）。

- **本文・UI（日本語対応）**: IBM Plex Sans JP（`--font-sans`）
- **見出しの editorial 一瞬**: Zen Old Mincho（`--font-serif` = `--font-display`）
- **数値・ID・ハッシュ・ラベル**: IBM Plex Mono（`--font-mono`）

### 4.1 セリフ（明朝）の適用範囲 — 限定する

CJK 明朝を密な UI の小サイズに広げると可読性が落ち、フォント重量も増す。よってセリフは **editorial の一瞬に限定**する。

| 用途 | 書体 |
|---|---|
| Display・表紙 | **serif**（Zen Old Mincho） |
| ページ H1 | **serif** |
| レコードタイトル（法令名・議案名・会議録名） | **serif** |
| H2 / H3・画面内見出し | sans（IBM Plex Sans） |
| パネル見出し | mono・大文字（§6） |
| 本文 | sans |
| 数値・金額・日時・ID・ハッシュ・ラベル | mono・tabular |

セリフは weight `600 / 700` のみ供給する（§11）。

### 4.2 スケールと規則

| 役割 | トークン | 規則 |
|---|---|---|
| Display | `--fs-display` 30px / 600 | serif |
| 見出し H1 | `--fs-h1` 22px / 700 | serif（ページ/レコードタイトル） |
| 見出し H2 | `--fs-h2` 17px / 600 | sans |
| 見出し H3 | `--fs-h3` 15px / 600 | sans |
| 本文 | `--fs-body` 15px / 400 | sans, line-height `--lh-body` 1.65 |
| 補助 | `--fs-sm` 13px / `--fs-xs` 12px | sans/mono |
| ラベル | `--fs-label` 11px | mono, 大文字, letter-spacing 0.14em |
| 数値 | mono | `font-variant-numeric: tabular-nums`（桁揃え） |

- 数値・金額・日時・ID・ハッシュは必ず等幅（mono ＋ tabular-nums）。
- ラベルは mono・大文字・字間広めで、本文と視覚的に区別する。

## 5. スペーシング・角丸

- 基本グリッド: **4px**。トークン `--s1 4`〜`--s8 64`（`4 / 8 / 12 / 16 / 24 / 32 / 48 / 64`）。
- 角丸（`ledger`・鋭い）: `--r-sm 2px`（小要素）, `--r 3px`（カード）, `--r-lg 4px`（パネル）, `--r-pill 999px`（pill）。
- 罫線は `--rule 1px` のヘアライン（`--hairline` / `--hairline-2` / `--hairline-3`）。構造は影でなく罫と余白で作る。
- 行・パネルの内余白は `--row-pad` / `--panel-pad`（regular 既定）。

## 6. コンポーネント仕様

機能要件のみ。スタイルはトークンに従う。

> **グリフ語彙の予約** — §3.3 の状態グリフ（`●◉Δ▲⊘`）は**状態の符号専用**。ブランドマークや装飾に流用しない（符号の過負荷を避ける）。ブランドは mono ワードマークなど、状態語彙と独立した表現を用いる。ナビ・UI ラベルに絵文字を使わない（無彩色基調 §1・プラットフォーム非一貫を避ける）。

- **Masthead（マストヘッド）** — 全ページ共通の最上段（layout 直下に1つ、全画面で不変）。`サイト識別（mono ワードマーク＋タグライン）＋ナビ（mono テキストラベル、絵文字不使用）` の識別段と、`スタンス（観測した事実を記録し、判断はしない）＋監視カバレッジ＋直近署名チェックポイント（`seq`＋`▸検証` 動線）` の来歴段から成る。**信頼を自称せず検証可能性を提示する**（USWDS banner の逆張り）。「稼働中」「検証済✓」など能動・自称の表示はしない（受動原則／レコード単位の検証バッジ禁止 ADR-000014）。来歴段の要素はデータが無い間は表示しない（嘘の値を出さない）。`--surface-1`＋下辺 `--hairline-2`。
- **Status pill** — 状態の表示。`色（低alpha背景＋枠）＋グリフ＋ラベル`。アイコンのみは不可（必ずラベルを伴う）。
- **Panel（パネル）** — 情報の最小単位。ヘッダ（左ドット＋mono大文字タイトル）＋本文。`--surface-1` ＋ `--hairline-2` ＋角丸 `--r-lg`、内余白 `--panel-pad`。複数を並べて1画面に統合する。
- **Metric tile（指標）** — ラベル（mono大文字）＋大きな数値（mono, tabular-nums）＋増減表示。**評価・優先順位を含意しない**（件数のみ＋「自動分類(未レビュー)」等の事実注記）。増減は中立に表す。
- **Change log item（変更タイムライン項目）** — 時刻（mono）＋状態ノード（色付き）＋本文＋ソース行（`SRC=…`・confidence・出典リンク）。状態ノードの枠色で changed/critical を区別。レコードタイトルは serif。
- **Diff（差分）** — 行頭記号 `+ / − / ~` ＋背景色 ＋左罫の三重符号化。背景は `--surface-inset` の well。法令XMLは条・項・号レベルの構造差分を優先（テキスト差分より高精度）。
- **Data table（テーブル）** — 高密度。ヘッダは mono大文字、数値列は右寄せ・tabular-nums。行ホバーは `--accent-weak`。
- **Alert（通知）** — アイコン（角丸の小バッジ）＋見出し＋本文＋左罫（caution/critical）。彩度を抑え、落ち着いたトーン。
- **Provenance chip（出所・鮮度）** — `最終取得時刻 / 出典 / 記録番号（安定した引用ハンドル）` を区切って表示。連結ハッシュの掲示や検証結果バッジは置かない（ADR-000014 / CONTEXT「連結」「完全性検証」）。
- **Map（地図）** — 選挙区レイヤー等の空間表示。**色は事実カテゴリのみ**（賛成多数/反対多数の別など）で、集計スコア・評価ではない。緑＝肯定のような価値含意を与えない（中立色）。マーカーは状態色＋形＋ラベル。`role="img"` ＋ `aria-label` を付す。比例選出は地図に乗らないため必ず横に件数で併記する（§5 非党派）。
- **Button** — primary（`--accent` 背景・文字 `--on-accent`）/ secondary（surface＋枠）/ ghost（透明）。
- **Focus** — `:focus-visible` で 2px 実線＋2px offset、`--focus`、≥3:1。

## 7. モーション

- 使ってよい: ロード時の段階的フェード（`animation-delay`）、ホバーの控えめな明度変化、状態遷移の短いフェード（120–200ms）。
- 避ける: 点滅（WCAG 2.3.1）、自動スクロール、点滅するグロー、視線を奪う動き。
- `prefers-reduced-motion: reduce` で全アニメーション・トランジションを停止する。

```css
@media (prefers-reduced-motion: reduce) {
  * { animation: none !important; transition: none !important; }
}
```

## 8. アクセシビリティ（WCAG 2.2 AA 目標）

- [ ] **コントラスト** — 本文 ≥ 4.5:1、大きい文字/非テキスト（UI要素・グラフ・フォーカス）≥ 3:1（1.4.3 / 1.4.11）。全色ペアを再計算する（ダークは自動で安全ではない）。`--text-faint` は装飾的メタ専用とし本文に使わない。
- [ ] **色だけに依存しない**（1.4.1）— 状態は色＋グリフ＋ラベル、差分は記号併用。
- [ ] **フォーカス可視**（2.4.7 / 2.4.11）— `:focus-visible` 2px＋offset、≥3:1。
- [ ] **OS設定の尊重** — `prefers-color-scheme` / `prefers-contrast` を尊重し、ユーザ選択を上書きしない。既定はダークだが、テーマ切替も提供する。
- [ ] **動きの抑制** — `prefers-reduced-motion` 対応。点滅不使用。
- [ ] **キーボード操作**（2.1.1）— 全機能をキーボードで操作可能に。スキップリンク、論理的な見出し階層、適切な ARIA。
- [ ] **無効状態** — 色だけで示さない（opacity ＋ cursor ＋ 表示の併用）。
- [ ] **標的サイズ**（2.5.8 / 2.5.5）— タッチ（`pointer: coarse`）では操作要素を **≥44×44px**。マウス等（`pointer: fine`）は高密度を保つ（§9）。
- [ ] **将来対応** — APCA（WCAG 3 草案、暗背景を非対称評価。本文 Lc 60+ 目安）を追加指標として併用。WCAG 2.x は引き続き遵守。

検証ツール: axe / WAVE / Lighthouse / Stark / APCA Contrast Calculator。

## 9. レスポンシブ／適応設計

**中核は意味論、密度は表現。** 凍結するのは「ダーク既定／色＝状態のみ＋グリフ＋ラベルの多重符号／純黒純白なし／provenance 常時／パネルが情報の最小単位／数値は mono・tabular」。§1 の「高密度でも疲れにくい」は High-Performance HMI の **デスクトップ表現** であり、画面幅・ポインタ種別に応じて呼吸してよい。狭幅では段階的開示でゆるめる。色・テーマ・状態の意味は決して作り替えない。

### 9.1 ブレークポイント（トークン化）

幅の閾値は役割名トークンとして `tokens.css` に集約する。最小対応幅は **360px**（360 / 390 で実機検証）。

```css
:root { --bp-sm:30rem; /*480*/ --bp-md:48rem; /*768*/ --bp-lg:55rem; /*880*/ --bp-xl:75rem; /*1200*/ }
```

### 9.2 container クエリ優先

「パネルが情報の最小単位」（§6）に従い、再利用されるパネル／カード／タイムライン行は **`@container`** で*自身の幅*に応じて折り返す（置かれたカラム文脈で自律的に振る舞う）。ページ骨格（topbar・地図グリッド等）だけ最小限の `@media` を使う。

- **タイムライン行（change log item）** — 狭幅 container では状態ノード（色＋グリフ＋ラベル）＋時刻を上部メタ行に集約し、タイトル／本文／chip／provenance を全幅で下へ縦積み。広幅は時刻・ノード・本文の多カラムを維持。
- **フィルタ** — 狭幅ではトグルで開閉（既定で閉、トグルに現在の絞り込み件数を併記）。主動線（タイムライン）を即座に見せる。明示的な開閉状態（`pointer: fine` / 広幅では常時展開）で実装する。

### 9.3 ポインタ適応の標的サイズ

```css
@media (pointer: coarse) {
  /* リンク・nav・ページャ・select/input/button・地図ズーム → ≥44×44px */
}
```

タッチでは快適さ（≥44px, WCAG 2.5.5 相当）、マウスでは高密度を両立する。

### 9.4 地図（cooperative gestures）

スマホでスクロールを奪わないため、対話型地図は **`cooperativeGestures`** を使う（1 本指＝ページスクロール／2 本指＝パン・ピンチズーム、ヒントは日本語）。高さは `clamp()` で狭幅は縮め、ポップアップはダークトークン（`--surface-3`/`--text-1`）で配色する。1 カラムに畳む際、**比例選出は地図直下に件数アンカー付きで併記し、決して画面外に追いやらない**（§5 非党派／CONCEPT §5）。

## 10. 声とコピー

いちばん大切な規定。**淡々と事実を記録する見張りの態度**を、文言で体現する。大仰・詩的・説得・煽動を避ける。生活者の言葉で、平易に、謙虚に。態度は宣言ではなく、画面全体の淡々とした佇まいで示す。

### 10.1 三つの構え

- **平易** — 生活者の言葉で。「改ざん検知用の値」（≒ `logHash`）のように、専門語に逃げず噛みくだく。
- **謙虚** — 自称しない。「検証済み」「稼働中」を名乗らない。信頼を求めず、確かめる手段を差し出す。
- **非判断** — 優先順位を指示しない。価値の含意を持つ色や語を避ける。事実とその出所だけを置く。

### 10.2 用語のレイヤリング（glossary との両立）

平易語と正規技術語を**層として併存**させる。`CONTEXT.md`（ubiquitous language）は不変。

| 文脈 | 使う語 |
|---|---|
| 生活者向け prose・CTA・見出し | 「**確かめる**」（例: この記録を確かめる）、「改ざん検知用の値」 |
| 正規技術語（glossary・ADR・技術説明・節タイトル） | 「**完全性検証**」「**署名チェックポイント検証**」「連結」 |
| マストヘッドの terse ラベル | `▸検証`（簡潔ラベルとして据え置き） |

例（検証パネル）:
- 見出し: 「**この記録を確かめる**」
- 説明: 「改ざん検知用の値をお使いのブラウザで計算し直し、記録済みの値と一致するか確かめます。」（内部的には**完全性検証**を当該端末で実行）

### 10.3 やめる／こうする

| ✗ やめる | なぜ | ✓ こうする |
|---|---|---|
| 「記録の連鎖を、あなたの端末で確かめる」 | 見出しが芝居がかっている | 「この記録を確かめる」 |
| 「改ざんがあれば連鎖は必ず破れます。確かめるのはあなたです。」 | 煽動・宣言 | 仕組みを平易に説明するだけ |
| 指標に「Δ 要確認」 | システムが優先順位を指示している | 件数のみ＋「自動分類(未レビュー)」 |
| 採決結果「可決」を緑で着色 | 緑＝肯定の価値含意 | 中立色＋「色は事実カテゴリのみ」と明記 |
| 各画面で「是非を判断しません」と繰り返す | 宣言自体が editorial・くどい | スタンスはマストヘッドの一行が一度だけ担う |

マストヘッド標語 ―― **観測した事実を記録し、判断はしない。**

## 11. フォント供給（セルフホスト）

フォントは**セルフホストする**。第三者（Google Fonts 等）へリクエストしない。

- **根拠** — 第三者 CDN は訪問者 IP を外部送出し、設計原則①（受動・sentinel）および「単一の自己ホストバイナリ」という目標アーキテクチャと相反する。
- **方式** — `@fontsource`（または静的 `@font-face`）で同梱し、`font-display: swap` を用いる。
- **weight-controlled** — CJK フォントは重い（weight あたり数 MB）。供給 weight を絞る:
  - mono（IBM Plex Mono）: 400 / 500 / 600
  - sans（IBM Plex Sans JP）: 400 / 500 / 600 / 700
  - serif（Zen Old Mincho）: **600 / 700 のみ**（§4 の限定適用に対応）
- **フォールバック** — 明朝ロード前は system 明朝（`Hiragino Mincho ProN` / `Yu Mincho` / `Noto Serif JP`）へフォールバックし、初回描画を阻害しない（`--font-serif` の stack 末尾に system 明朝を持つ）。
